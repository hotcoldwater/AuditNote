GamsaNote 최종 기획안
1. 서비스 개요

GamsaNote는 회계감사 기준서 제목을 보고 내용을 직접 작성하면서, 기준서 문구와 구조를 반복 암기하는 웹 기반 학습 서비스다.

사용자는 기준서 제목만 보고 답안을 직접 작성하고, 서비스는 사용자의 답안을 실제 기준서 내용과 비교하여 키워드 기반으로 채점한다. 점수가 낮거나 사용자가 불안하다고 판단한 항목은 오답노트에 저장되어 반복 학습할 수 있다.

GamsaNote의 핵심 목적은 단순 문제풀이가 아니라, 회계감사 기준서 문구와 구조를 시험장에서 직접 떠올릴 수 있을 정도로 반복 훈련하는 것이다.

2. 서비스 한 줄 정의

GamsaNote는 회계감사 기준서 제목을 보고 답안을 직접 작성하고, 키워드 기반 채점과 오답 반복을 통해 기준서 암기를 훈련하는 웹앱이다.

3. 서비스 핵심 방향

GamsaNote는 10문제, 20문제처럼 문제 세트를 완주하는 테스트 앱이 아니다.

사용자가 출퇴근 시간, 이동 중, 쉬는 시간에 모바일로 짧게 사용할 수 있도록 1문제씩 계속 출제되는 연속 학습 방식을 사용한다.

기본 흐름은 다음과 같다.

기준서 제목 확인
→ 사용자가 직접 답안 작성
→ 제출
→ 즉시 채점
→ 결과 확인
→ DB 저장
→ 다음 문제
→ 언제든 종료 가능

사용자는 한 문제만 풀고 앱을 종료해도 된다.

모든 풀이 기록은 문제 제출 시점마다 Supabase DB에 저장된다.

주제별 학습은 편 단위로만 선택한다.

장 단위 선택은 제외한다.

이유는 다음과 같다.

장 단위는 범위가 너무 좁아진다.
모바일 짬공부 목적에 맞지 않는다.
선택 과정이 복잡해진다.
편 단위 선택만으로도 충분히 주제별 학습 효과가 있다.

주제별 학습 흐름은 다음과 같다.

학습 시작
→ 주제별 선택
→ 1편~6편 중 하나 선택
→ 해당 편 전체에서 문제 1개 출제
→ 풀이
→ 결과 확인
→ 다음 문제

사용자는 언제든지 학습을 종료할 수 있다.

중요 원칙은 다음과 같다.

매 문제 제출 시마다 Supabase DB에 즉시 저장한다.

10. LV 기반 중요도 체계

현재 버전에서는 출제빈도를 사용하지 않는다.

중요도와 출제 확률은 LV만으로 구성한다.

Lv1: 필수. 반드시 암기해야 하는 핵심 항목.

Lv2: 필수. 우선적으로 반복해야 하는 기본 항목.

Lv3: 필수. 실전 대비 기본 암기 항목.

Lv4: 심화. 중요하지만 난도가 있거나 분량이 있는 항목.

Lv5: 지엽. 우선순위는 낮지만 고득점 대비용 항목.

사용자가 난이도를 직접 선택하지 않는다.

앱이 자동으로 LV와 사용자 풀이 이력을 반영하여 문제를 출제한다.

11. 문제 출제 확률 로직
11-1. 기본 LV 가중치

MVP 기본값은 다음과 같이 둔다.

Lv1 = 10

Lv2 = 10

Lv3 = 10

Lv4 = 5

Lv5 = 2

즉, Lv1~3은 가장 높은 확률, Lv4는 중간 확률, Lv5는 낮은 확률로 출제된다.

11-2. 사용자 풀이 이력 보정

최종 출제 가중치는 다음 방식으로 계산한다.

최종 가중치 = LV 기본 가중치 × 학습상태 보정계수

학습상태 보정계수는 다음과 같다.

미풀이: 1.0
아직 안 푼 문제이므로 기본 확률 유지

최근 오답: 0.8
다시 나오되 전체 회독을 방해하지 않음

오답 경험 있음: 0.7
정답 문제보다는 자주 나옴

REVIEW: 0.6
애매하게 맞힌 문제

CORRECT: 0.3
맞힌 문제는 확률 크게 감소

EXCELLENT: 0.15
충분히 아는 문제는 매우 낮게

핵심 원칙은 다음과 같다.

미풀이 문제를 우선적으로 본다.

맞힌 문제는 출제 확률을 크게 낮춘다.

틀린 문제도 출제 확률을 낮추지만, 맞힌 문제보다는 덜 낮춘다.

전체 기준서를 골고루 회독할 수 있게 한다.

12. 채점 방식

MVP에서는 AI 채점을 사용하지 않는다.

첫 버전에서는 키워드 기반 채점을 사용한다.

총점은 100점이다.

필수 키워드: 50점
반드시 들어가야 하는 핵심 표현

선택 키워드: 20점
있으면 좋은 보조 표현

답안 충실도: 15점
답안 길이와 성실성

원문 유사도: 15점
정답 원문과의 단순 텍스트 유사도

합계: 100점

13. 필수 키워드 점수

requiredKeywords에 있는 단어가 사용자 답안에 얼마나 포함되었는지 확인한다.

계산식은 다음과 같다.

필수 키워드 점수 = 50 × 포함한 필수 키워드 수 / 전체 필수 키워드 수

예시:

필수 키워드: 적격성, 능력, 객관성, 감사증거

사용자 답안에 포함된 키워드: 적격성, 능력, 감사증거

점수 = 50 × 3 / 4 = 37.5점

14. 선택 키워드 점수

optionalKeywords에 있는 단어가 사용자 답안에 얼마나 포함되었는지 확인한다.

계산식은 다음과 같다.

선택 키워드 점수 = 20 × 포함한 선택 키워드 수 / 전체 선택 키워드 수

선택 키워드는 들어가면 좋지만, 없다고 해서 바로 오답으로 보지는 않는다.

15. 답안 충실도 점수

사용자 답안 길이와 정답 원문 길이를 비교한다.

계산식은 다음과 같다.

답안 길이 비율 = 사용자 답안 글자 수 / 정답 원문 글자 수

점수는 다음과 같이 부여한다.

80% 이상: 15점

60~79%: 12점

40~59%: 8점

20~39%: 4점

20% 미만: 0점

이 항목은 사용자가 키워드만 단순 나열해서 높은 점수를 받는 것을 방지하기 위한 장치다.

16. 원문 유사도 점수

정답 원문과 사용자 답안의 단순 텍스트 유사도를 계산한다.

MVP에서는 복잡한 의미 분석을 하지 않는다.

정규화 기준은 다음과 같다.

공백 제거

문장부호 제거

특수문자 제거

대소문자 통일

원문 유사도는 최대 15점이다.

17. 캡 규칙

단순 합산만으로 이상한 점수가 나오지 않도록 제한 규칙을 둔다.

필수 키워드 포함률 50% 미만: 최대 59점

답안 길이가 정답 원문의 20% 미만: 최대 50점

미응답 / 모르겠어요: 0점

필수 키워드 100% 포함 + 답안 충실도 40% 이상: 최소 75점 보장 가능

31. GamsaNote의 핵심 철학

GamsaNote는 문제 세트를 완주하는 앱이 아니다.

GamsaNote는 사용자가 짧은 시간에 한 문제씩 기준서를 떠올리고, 매번 기록을 남기며, 전체 범위를 자연스럽게 회독하도록 돕는 회계감사 암기 루프 앱이다.

핵심은 채점의 완벽함보다 반복해서 떠올리게 만드는 구조다.

따라서 첫 버전에서는 복잡한 AI 채점보다 다음 흐름을 우선한다.

기준서 제목
→ 직접 작성
→ 정답 원문 확인
→ 빠진 키워드 확인
→ 오답 반복
→ 누적 기록 시각화
